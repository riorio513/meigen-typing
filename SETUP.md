# ログイン機能のセットアップ手順

このアプリはFirebase(Googleの無料サービス)を使って、メールアドレスでのログインと
タイピング成績の保存を行います。以下の手順を一度だけ行えば、どのPCからログインしても
同じ成績が表示されるようになります。

## 1. Firebaseプロジェクトを作成する

1. [Firebaseコンソール](https://console.firebase.google.com/) にGoogleアカウントでアクセス
2. 「プロジェクトを追加」をクリックし、プロジェクト名を入力(例: meigen-typing)
3. Googleアナリティクスは「有効にしない」でOK
4. プロジェクトが作成されるまで待つ

## 2. メールリンクログインを有効にする

1. 左メニューの「構築」→「Authentication」を開く
2. 「始める」をクリック
3. 「Sign-in method」タブで「メールリンク(パスワードなしでログイン)」を選択して有効化

## 3. Firestore Database を作成する

1. 左メニューの「構築」→「Firestore Database」を開く
2. 「データベースの作成」をクリック
3. ロケーションは `asia-northeast1(東京)` など好きな場所を選択
4. 「本番環境モードで開始」を選択

## 4. Firestoreのセキュリティルールを設定する

「ルール」タブを開き、以下の内容に置き換えて「公開」をクリックしてください。
(これは「自分のデータは自分だけが読み書きできる」というルールです)

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /typingStats/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

## 5. ウェブアプリを登録して設定値を取得する

1. 左上の歯車アイコン→「プロジェクトの設定」を開く
2. 「マイアプリ」欄の `</>` (ウェブ) アイコンをクリック
3. アプリのニックネームを入力して「アプリを登録」
4. 表示された `firebaseConfig` の中身(`apiKey`, `authDomain` など)をコピー
5. このフォルダにある `firebase-config.js` を開き、`YOUR_API_KEY` などの部分を
   コピーした値に書き換えて保存する

## 6. 公開ドメインを承認済みドメインに追加する

GitHub Pagesなどで公開した後、そのURLのドメイン(例: `yourname.github.io`)を
「Authentication」→「Settings」タブ→「承認済みドメイン」に追加してください。
これを忘れるとログインリンクが正しく機能しません。

## 7. 動作確認

1. 公開したページを開き、メールアドレスを入力して「ログインリンクを送る」をクリック
2. 届いたメールのリンクを開く(同じPCでもOK)
3. ログインできれば、タイピング終了時に累計文字数が保存され、
   別のPCで同じメールアドレスでログインしても続きの成績が表示されます

## 困ったときは

- ログイン画面にエラーメッセージが出たら、内容に沿って設定を見直してください
- 「Firebaseの設定が正しくありません」と出る場合は `firebase-config.js` の値を再確認
- 「承認済みドメイン」への追加を忘れていないか確認してください
